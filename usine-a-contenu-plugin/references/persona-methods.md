# Persona — Méthodes de traitement par source

Référence lue par `persona-builder`.

---

## Transcriptions d'appels / interviews

**Ce qu'on cherche :** formulations exactes, problèmes décrits dans leurs propres mots, objections spontanées, résultats attendus.

**Traitement :**
1. Lire intégralement (ou extraits collés)
2. Identifier les patterns récurrents sur 3+ occurrences
3. Extraire les verbatims les plus représentatifs (citation directe de préférence)
4. Repérer les contradictions ou nuances (ce qu'ils disent vs. ce qu'ils font)

**Signal fort :** une frustration exprimée spontanément, pas en réponse à une question fermée.

---

## Avis clients (Google, Trustpilot, Amazon, etc.)

**Ce qu'on cherche :** vocabulaire utilisé pour décrire les bénéfices, les frustrations avant achat, les déclencheurs de la décision.

**Traitement :**
1. Analyser 20 à 50 avis minimum
2. Repérer les mots revenant dans 30%+ des avis positifs
3. Repérer les mots revenant dans les avis négatifs
4. Identifier le "avant / après" : que faisaient-ils avant, qu'ont-ils obtenu ?
5. Extraire 3 verbatims représentatifs (avis 3-4 étoiles souvent les plus nuancés)

---

## Reddit / forums / commentaires

**Ce qu'on cherche :** questions récurrentes, vocabulaire non filtré, objections et doutes.

**Traitement :**
1. Chercher les posts avec le plus d'upvotes et de commentaires sur le sujet
2. Lire les commentaires les plus longs (souvent les plus riches)
3. Identifier les questions posées plusieurs fois
4. Repérer le ton : sarcastique, frustré, enthousiaste, sceptique
5. Extraire les formulations qui reviennent

---

## Interviews ou formulaires

**Ce qu'on cherche :** même chose que transcriptions, mais attention aux biais de réponse désirable.

**Traitement :**
1. Prioriser les questions ouvertes sur les questions fermées
2. Comparer les réponses entre personas différents
3. Méfiance sur les réponses trop positives ou trop génériques

---

## Recherche web

**Ce qu'on cherche :** comportements de recherche, questions fréquentes, intention réelle.

**Traitement :**
1. Analyser les suggestions de recherche (Google, YouTube)
2. Identifier les questions dans les PAA ("People also ask")
3. Regarder les forums et Reddit qui remontent
4. Utiliser les titres des contenus les plus partagés pour déduire l'angle qui résonne

---

## Description intuitive

**Utilisation :** quand aucune donnée n'est disponible. Le persona intuitif est un point de départ à valider, pas une vérité.

**Traitement — 5 questions guidées :**
1. Quel est son problème principal en ce moment ?
2. Qu'a-t-il déjà essayé pour le résoudre ?
3. Pourquoi ça n'a pas marché ?
4. Qu'est-ce qui l'empêche d'agir ?
5. Quel résultat concret le ferait passer à l'action ?

Annoter la fiche avec : `[Source : intuitif — à valider]`
